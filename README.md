# vibescan

Agent-readiness scanner + AI/agent footprint recon. Single-file Python script,
no external dependencies (stdlib only).

Built on top of `agent_ready.py` (a console port of the checks performed by
[isitagentready.com](https://isitagentready.com)) and extended with two extra
layers for offensive recon: leaked AI agent context files and vibecoding
fingerprints.

## What it does

Three layers per scan:

1. **Readiness** *(scored)* ÔÇö robots.txt, Sitemap, Link headers, DNS-AID,
   Markdown negotiation, AI bot rules, Content Signals, Web Bot Auth,
   API Catalog, OAuth/OIDC discovery, OAuth Protected Resource, Auth.md,
   MCP Server Card, Agent Skills index, WebMCP, plus the optional Commerce
   bucket (x402, MPP, UCP, ACP).
2. **Exposures** *(not scored)* ÔÇö leaked files commonly produced by code
   agents: `CLAUDE.md`, `AGENTS.md`, `.cursorrules`, `.windsurfrules`,
   `.aider.*`, `.github/copilot-instructions.md`, `.continue/`, `.claude/`,
   `.cursor/`, `.codex/`, `.specstory/`. Also the classics: `.env*`, `.git/*`,
   SQL dumps, archive backups, IDE configs, lockfiles, public `openapi.json`.
3. **Fingerprint** *(not scored)* ÔÇö vibecoding signals: Lovable, v0.dev,
   bolt.new, Base44, GPT-Engineer, Replit, generic "made with ÔÇĞ" and
   "generated by ÔÇĞ" watermarks, builder-hosted suffixes, Supabase anon JWTs
   embedded in bundles.

Output: a flat stream of findings (ProjectDiscovery-style), then three
summary blocks at the end: `[VIBECODING]` verdict, `[EXPOSURES]` count, and
`[SCORE]` agent-readiness with grade and tier.

## Install

```bash
git clone https://github.com/openbashok/vibescan.git
cd vibescan
```

Python 3.10+. No `pip install` needed.

## Usage

### Single host

```bash
python3 vibescan.py openbash.com
```

### Multi-host

From a file (one target per line, `#` comments allowed):

```bash
python3 vibescan.py -l hosts.txt
```

Inline:

```bash
python3 vibescan.py openbash.com example.com api.target.tld
```

From stdin:

```bash
echo "openbash.com" | python3 vibescan.py -l -
```

### Common flags

| Flag | Effect |
|---|---|
| `--json` | JSON output (single object for one target, array of objects for many) |
| `-v`, `--verbose` | Show every check in `exposures`/`fingerprint`, including misses |
| `--only LAYER` | Run only specific layers (repeatable: `--only readiness --only fingerprint`) |
| `--no-score` | Suppress the `[SCORE]` block |
| `--no-verdict` | Suppress the `[VIBECODING]` block |
| `--no-follow` | Do not follow initial redirects (scan the literal host) |
| `--width N` | Fix column width (default: terminal width) |
| `--no-color` | Disable ANSI colors |

## Scoring

The readiness score follows the same formula used by isitagentready.com:

```
overall = (╬ú passes across scored checks) / (╬ú scored checks) ├ù 100
```

Content Signals and Commerce checks are displayed but do not enter the
score. WebMCP counts as `fail` when not detected via HTTP (we cannot run a
headless browser).

| Score | Grade | Tier |
|---|---|---|
| 0ÔÇô19 | F | Pre-Agent |
| 20ÔÇô39 | D | Crawlable |
| 40ÔÇô59 | C | Discoverable |
| 60ÔÇô79 | B | Negotiable |
| 80ÔÇô94 | A | Agentic |
| 95ÔÇô100 | A+ | Native |

## Vibecoding verdict

Independent of the readiness score. Weighted sum of signals that indicate
the site was built with or for AI coding agents:

- Exposed agent context files (CLAUDE.md, .cursorrules, ÔÇĞ): 12ÔÇô15 pts each
- Builder watermarks (Lovable, v0, Bolt, Base44, ÔÇĞ): 8ÔÇô15 pts each
- Builder-hosted suffix (lovableproject.com, bolt.host, base44.app, ÔÇĞ): 2ÔÇô10 pts
- `<meta generator>` matching a builder: 8 pts

Verdict labels: `not detected` (0), `weak signals` (1ÔÇô9), `likely vibecoded`
(10ÔÇô29), `vibecoded confirmed` (30+).
